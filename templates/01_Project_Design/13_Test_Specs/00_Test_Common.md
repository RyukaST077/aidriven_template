## 2. テストレベル定義
| レベル | 略称 | 内容 | 実施者 |
| -- | -- | -- | -- |
| 単体テスト | UT | 関数・メソッド単位の動作確認 | 開発者 |
| 結合テスト | IT | 機能間・画面間の連携確認 | 開発者/テスター |
| システムテスト | ST | 業務シナリオに基づく通し確認 | テスター/QA |

## 3. 合否判定基準
- 全テストケースが実施済みであること
- 重要度「高」のバグが残存していないこと
- 既知のバグについては回避策や対応方針により合意が取れていること
