## 1. 目的・スコープ

### 1.1 目的

* 本書で定義すること：

  * To-Beの業務フロー（標準・例外・分岐）
  * 業務ルール（判断基準、計算、状態遷移、締め、承認、権限）
  * ルールを支えるデータ、画面操作、外部連携、バッチの前提
* 本書で定義しないこと（他ドキュメントへ委譲）：

  * 画面レイアウト詳細 → 画面設計書
  * API/IF詳細 → 外部IF設計書
  * テーブル物理定義 → データ設計書

### 1.2 対象業務範囲

* 対象業務領域：日程調整（候補提示、回答収集、集計、最適候補提示、確定、通知、リマインド、再調整）
* 対象組織／拠点：社内チーム/部署、または友人グループ等の小〜中規模コミュニティ
* 対象ユーザー種別：幹事（主担当/共同幹事）、参加者、運用者、組織管理者
* 対象チャネル（Web、モバイル、EDI等）：TBD

* マスタの権威／更新責任：TBD

## 2. 全体像（サマリ）

### 2.1 業務ゴールとKPI（任意）

* ゴール：日程確定までのリードタイム短縮と幹事の作業削減
* 指標：
  * 日程確定までのリードタイム中央値 50%短縮（例：7日→3日）
  * 幹事の作業時間 40%削減
  * 参加者の回答率（締切まで）90%
  * 幹事手動のリマインド回数 0〜1回

### 2.2 To-Be業務プロセス一覧（ドラフト）

| プロセスID | プロセス名 | 開始条件 | 終了条件 | 主要ロール | 関連機能（FR-ID） | 優先度 | 備考 |
| ------ | ----- | ---- | ---- | ----- | ---------- | --- | -- |
| BP-001 | イベント作成・招待 | 幹事がイベント作成を開始 | 招待リンク発行・参加者に通知 | 幹事 |  | Must |  |
| BP-002 | 回答収集 | 参加者が招待を受領 | 締切到来または全員回答 | 参加者 |  | Must |  |
| BP-003 | 集計・最適候補提示 | 回答が集計可能状態 | 幹事が候補を比較できる | 幹事 |  | Must |  |
| BP-004 | 確定・通知 | 幹事が確定操作 | 参加者へ確定通知送信 | 幹事 |  | Must |  |
| BP-005 | リマインド・再調整 | 締切前または不足回答 | 期限延長/候補追加/再通知 | 幹事 |  | Should |  |

| BP-011 | 管理者設定 | 組織管理者が設定操作 | 設定反映完了 | 組織管理者 |  | Must |  |

### 2.3 状態遷移（高レベル）

* 対象：イベント（調整）
* 状態一覧（案）：下書き／公開中／締切前／締切／再調整／差戻し／確定／キャンセル
* 遷移の概要：締切到来で自動的に「締切」へ遷移。全員回答時は幹事へ確定促進通知。

> 補足：詳細な状態遷移表は「5. 業務ルール（状態・承認・締め）」で定義。

### 3.2 権限の粒度（方針）

* 権限判定単位：

  * 機能（メニュー／API）
  * 操作（参照／登録／更新／削除／承認／取消／出力／再処理等）
  * データ範囲（自組織のみ、担当イベントのみ 等）
* 認可方式：RBAC
* 代行・委任：必要（期間指定・記録必須）
* 認証/本人確認方針：幹事はログイン必須、参加者は招待リンクでゲスト回答可（必要に応じてOTP等で本人確認）

### 5.3 ルール詳細（主要BRL）

#### BRL-001：最適候補提示（必須参加者優先）

* 分類：BRL-A
* 目的：必須参加者が全員参加可能な候補を優先して提示する
* 適用範囲：BP-003（集計・最適候補提示）
* 対象データ：候補日時、回答（可/不可/未定）、必須参加者フラグ
* 入力：候補別の回答集計、必須参加者リスト
* 出力：候補の優先順位
* ルール本文：必須参加者が全員「可」の候補が存在する場合、それらを優先表示し、同条件内では可人数が最大の候補を上位とする
* 例外・除外条件：必須参加者が全員「可」の候補が存在しない場合は、全候補を対象に可人数最大で順位付けする
* 関連ルール：BRL-002

##### 5.3.1 決定表（Decision Table）

| 条件                 | C1 | C2 |
| ------------------ | -- | -- |
| 必須参加者が全員「可」の候補が存在 | Y  | N  |
| **結果：候補の対象集合** | 必須全員可の候補のみ | 全候補 |
| **結果：順位付け**        | 可人数最大→BRL-002 | 可人数最大→BRL-002 |

#### BRL-002：候補順位タイブレーク

* 分類：BRL-A
* 目的：同条件の候補が複数ある場合の順位付け基準を明確化する
* 適用範囲：BP-003
* 入力：候補別の可人数、未定人数、候補日時
* 出力：候補の順位
* ルール本文：可人数が同一の場合、未定人数が少ない候補を優先し、それでも同じ場合は日時が早い候補を優先する

##### 5.3.1 決定表（Decision Table）

| 条件                 | C1 | C2 | C3 |
| ------------------ | -- | -- | -- |
| 可人数が同一            | Y  | Y  | Y  |
| 未定人数が少ない        | Y  | N  | N  |
| 日時が早い             | -  | Y  | N  |
| **結果：優先候補**       | 未定少の候補 | 日時が早い候補 | 同順位（幹事判断） |

#### BRL-003：締切の初期値と必須設定

* 分類：BRL-T
* 目的：締切未設定を防止し、既定値で初期化する
* 適用範囲：BP-001
* 入力：幹事が設定した締切有無
* 出力：締切日時

| **結果：締切日時**       | 入力値 | 作成日時+3日 18:00 |

* 分類：BRL-T
* 目的：未回答者に対する自動リマインドの送信条件を明確化する
* 適用範囲：BP-005
* 入力：締切までの残時間、未回答者有無、幹事のON/OFF設定
